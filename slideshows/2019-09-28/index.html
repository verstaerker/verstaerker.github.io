The media query defines a condition. The first source, on which it is truthy, will be used.

<picture>

      <picture>
        <source media="(min-width: 650px)" srcset="large.jpg">
        <source media="(min-width: 465px)" srcset="small.jpg">
        <img src="fallback.jpg" alt="Flowers">
      </picture>

The srcset defines the source image.

<picture>

      <picture>
        <source media="(min-width: 650px)"
                srcset="large.jpg 1x, large@2x.jpg 2x">
        <source media="(min-width: 465px)" srcset="small.jpg">
        <img src="fallback.jpg" alt="Flowers">
      </picture>

It can also be used to serve multiple pixel densities.

<picture>

      <picture>
        <source media="(min-width: 650px)" srcset="large.jpg">
        <source media="(min-width: 465px)" srcset="small.jpg">
        <img src="fallback.jpg" alt="Flowers">
      </picture>

The <img> element is used to display the evaluated or fallback image.

<picture>

      <picture>
        <source srcset="exotic.jp2">
        <source srcset="modern.webp">
        <img src="legacy.jpg" alt="Flowers">
      </picture>

<img srcset>

    <img srcset="small.jpg 320w, large.jpg 1440w, fallback.jpg"
         src="fallback.jpg">

With srcset we can define a set of resolutions for the same image. The images should have the same ration. "w" (width) equals "px".

<img srcset>

    <img srcset="small.jpg 320w, large.jpg 1440w, fallback.jpg"
         src="fallback.jpg">

The src value is used as a fallback.

<img srcset>

    <img srcset="small.jpg 320w, large.jpg 1440w, fallback.jpg"
         sizes="(max-width: 1440px) 50vw, 100vw"
         src="fallback.jpg">

With sizes we can define how the image is displayed in the layout. Here the image uses 50% (50vw) of screens up to 1440px. Then the image is displayed in full width.
